Portada / Introducción

La recursividad es una técnica en la que una función se llama a sí misma para resolver instancias más pequeñas del mismo problema. Es fundamental en ciencias computacionales por su elegancia en la expresión de soluciones a problemas tales como búsqueda, división y conquista, estructuras fractales y problemas combinatorios.

Definición Aplicaciones Ejemplos Conclusión

Definición de recursividad

Un procedimiento recursivo se define por: (1) los casos base que terminan la recursión, y (2) el paso recursivo que reduce el problema hacia el caso base. Sin un caso base, la recursión no termina.

Procedimientos recursivos y aplicaciones

  • Búsqueda y ordenamientos (ej. mergesort, quicksort).
  • Algoritmos combinatorios y generación de permutaciones.
  • Problemas clásicos: Factorial, Fibonacci, Torres de Hanoi.
  • Generación de fractales y estructuras auto-similares.

Características de la recursividad

  • Expresividad: permite soluciones compactas y legibles.
  • Uso de pila de llamadas: cada llamada crea un marco con parámetros y variables locales.
  • Riesgo de desbordamiento de pila si la profundidad es muy grande.
  • Puede ser optimizada con técnicas como memoización o transformación a iterativa.

Complejidad computacional

El análisis de recursión se hace mediante ecuaciones de recurrencia. Para muchos algoritmos divide‑y‑vencerás se emplea el Teorema Maestro (Master Theorem) para obtener la complejidad asintótica.

Ejemplos con visuales

Factorial (recursivo)

// JavaScript - factorial recursivo
function fact(n){
  if(n<=1) return 1;
  return n * fact(n-1);
}

Complejidad: O(n) llamadas recursivas, O(n) tiempo y O(n) espacio en pila.

Fibonacci (recursivo)

// Recursivo sencillo (ineficiente)
function fib(n){
  if(n<=1) return n;
  return fib(n-1)+fib(n-2);
}

Complejidad: O(φ^n) (exponencial) en la versión pura; se mejora con memoización a O(n).

Torres de Hanoi

// Mover n discos de A a C usando B
function hanoi(n, A, B, C){
  if(n==0) return;
  hanoi(n-1, A, C, B);
  move(A, C);
  hanoi(n-1, B, A, C);
}

Complejidad: T(n)=2T(n-1)+1 ⇒ T(n)=2^n − 1 (exponencial).

Fractales (Sierpinski)

El triángulo de Sierpinski se puede generar aplicando recursión en subdivisiones; también aparece si coloreamos impares en el triángulo de Pascal.

Conclusión

La recursividad es una herramienta poderosa que simplifica el diseño de algoritmos y modela naturalmente problemas auto‑similares. Sin embargo, debe usarse con conocimiento de su coste en tiempo y espacio, y combinada con técnicas (memoización, optimización de cola) cuando sea necesario.
